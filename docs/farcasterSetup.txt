## How to Generate a Real Farcaster Account Association for a Mini App Manifest

The `accountAssociation` in `/.well-known/farcaster.json` cryptographically proves that a Farcaster account (identified by an FID) is the verified owner/author of a given domain. It is based on the **JSON Farcaster Signature (JFS)** standard.

---

### What it is

It's a signed message where a Farcaster account signs a payload containing only your app's domain. The signature is produced by the account's **custody address** (Ethereum wallet) or a valid **auth address**.

---

## Step-by-Step Process

### Method 1: Using the Farcaster Developer Tool (Recommended / Easiest)

This is the **official, no-code way** and what `https://farcaster.xyz/~/developers/mini-apps/manifest` is for.

#### Step 1 — Enable Developer Mode
1. Log in to Farcaster (web or mobile)
2. Go to `https://farcaster.xyz/~/settings/developer-tools`
3. Toggle on **"Developer Mode"**
4. A developer section appears in the left sidebar

#### Step 2 — Open the Manifest Tool
Navigate to: `https://farcaster.xyz/~/developers/mini-apps/manifest`

#### Step 3 — Enter Your Domain
- Enter the **exact FQDN** (fully qualified domain name) of where your app is hosted
- No protocol (`https://`), no path, no port — just the bare domain
  - ✅ `myapp.example.com`
  - ❌ `https://myapp.example.com`
  - ❌ `myapp.example.com/app`

#### Step 4 — Sign with Your Wallet
The tool triggers a **wallet signature request** on your connected Farcaster custody or auth address. You approve the signature in your wallet.

> ⚠️ **The domain you sign must exactly match where you host the file.** `www.example.com` and `example.com` are treated as different apps.

#### Step 5 — Receive the Output
The tool returns three base64-encoded strings: `header`, `payload`, `signature`.

---

### Method 2: Using `sdk.experimental.signManifest()` (Programmatic)

Available in `@farcaster/miniapp-sdk`. This is an **experimental** SDK action (may change or be removed):

```ts
import { sdk } from '@farcaster/miniapp-sdk'

const result = await sdk.experimental.signManifest({
  domain: 'example.com'
})
// result = { header: string, payload: string, signature: string }
```

**Caveats:**
- This only works when your mini app is **running inside a Farcaster host client** that supports it
- Host behavior varies: some hosts may restrict signing to allowlisted domains, or disable the feature entirely
- Throws `SignManifest.RejectedByUser`, `SignManifest.InvalidDomain`, or `SignManifest.GenericError`

---

## What Data Is Returned

Both methods return the same three fields, which are the three components of a **JSON Farcaster Signature (JFS)**:

| Field | Type | Description |
|---|---|---|
| `header` | `string` | Base64url-encoded JSON object containing `fid`, `type`, and `key` |
| `payload` | `string` | Base64url-encoded JSON object containing just `{ "domain": "your-domain.com" }` |
| `signature` | `string` | Base64url-encoded cryptographic signature over `header.payload` |

### Decoded `header` example:
```json
{
  "fid": 3621,
  "type": "custody",
  "key": "0x2cd85a093261f59270804A6EA697CeA4CeBEcafE"
}
```
- `fid`: Your Farcaster account ID
- `type`: `"custody"` (signed by custody wallet) or `"auth"` (signed by an auth address)
- `key`: The Ethereum address that produced the signature

### Decoded `payload` example:
```json
{
  "domain": "yoink.party"
}
```
Only the domain — nothing else.

### `signature`:
EIP-191 personal signature (or equivalent), covering the string `header.payload` (the two base64 strings joined by a dot).

---

## How to Use header, payload, signature in the Manifest JSON

Place all three values under the top-level `accountAssociation` key in `/.well-known/farcaster.json`:

```json
{
  "accountAssociation": {
    "header": "eyJmaWQiOjM2MjEsInR5cGUiOiJjdXN0b2R5Iiwia2V5IjoiMHgyY2Q4NWEwOTMyNjFmNTkyNzA4MDRBNkVBNjk3Q2VBNENlQkVjYWZFIn0",
    "payload": "eyJkb21haW4iOiJ5b2luay5wYXJ0eSJ9",
    "signature": "D7urCIelYBtFc12UkEw/VOE1rkGKMdmDqAP/6s7sLWNfjhyug8pQCTM68XVJ8Gal6eBZxvtFE4RpVwqDtnLrzhs="
  },
  "miniapp": {
    "version": "1",
    "name": "Your App Name",
    "iconUrl": "https://yourapp.com/icon.png",
    "homeUrl": "https://yourapp.com",
    "splashImageUrl": "https://yourapp.com/splash.png",
    "splashBackgroundColor": "#ffffff",
    "webhookUrl": "https://yourapp.com/api/webhook",
    "subtitle": "Short tagline",
    "description": "What your app does.",
    "primaryCategory": "social"
  }
}
```

This file must be served at `https://yourdomain.com/.well-known/farcaster.json`.

## Alternative: Farcaster Hosted Manifests

Instead of self-hosting, Farcaster can host the manifest for you:

1. Go to `https://farcaster.xyz/~/developers/mini-apps/manifest`
2. Enter your domain + app details → signing happens in the tool
3. You receive a **hosted manifest ID**
4. Set up a redirect in your server:

```ts
// Next.js example
redirects: [
  {
    source: '/.well-known/farcaster.json',
    destination: 'https://api.farcaster.xyz/miniapps/hosted-manifest/YOUR_ID',
    permanent: false,
  }
]
```

This means you never need to touch the manifest file again — update everything from the Farcaster developer tools UI.

## Summary

| Step | What happens |
|---|---|
| 1. Open manifest tool | `farcaster.xyz/~/developers/mini-apps/manifest` |
| 2. Enter exact domain | Must match hosting domain exactly |
| 3. Sign with wallet | Custody or auth address signs `{ "domain": "..." }` |
| 4. Get JFS output | Three base64 strings: `header`, `payload`, `signature` |
| 5. Place in manifest | Under `accountAssociation` in `/.well-known/farcaster.json` |
| 6. Verify | Green checkbox appears in manifest tool = verified ✅ |

The `accountAssociation` is what links your Farcaster FID to your domain — without it, your app cannot be registered, added by users, or send notifications.
